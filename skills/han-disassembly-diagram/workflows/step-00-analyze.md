# Step 0: 防幻觉分析

先建目录、识别对象、分析结构。产物是提示词的「事实底座」。

## 0.1 建工作目录

- 从请求**识别目标对象**(替换 `{……}` 占位为具体对象)。
- 从对象名派生 slug(kebab-case)。
- 在用户当前工作目录建:
  ```
  disassembly-diagram/{slug}/
  ├── source-{slug}.md
  ├── analysis.md
  ├── structured-content.md
  ├── refs/
  └── prompts/
  ```

## 0.2 写 analysis.md

- 标题 + 目标对象
- 受众 + 用途(课堂/科普文章/社交媒体封面/知识卡)
- 推荐参数(aspect/mode 初步,Step 1 确认)
- 要展示的**外观特征**
- 可能的**内部结构**与组件组
- **材料类别**:金属/塑料/玻璃/橡胶/陶瓷/织物/复合/流体/电子模块
- **工作原理**序列(能量输入 → 控制 → 核心运行 → 输出)
- **不确定性说明**

> **高价值对象**(品牌产品/医疗器械/安全设备/武器等):结构须基于用户提供的权威资料;确切内部未知时,标注为「示意图解说」而非精确工程拆解。

## 0.3 写 structured-content.md(简体中文)

- 标题 + 副标题
- 5-8 个主要外/内部标签
- 3-6 个爆炸视图组件标签
- 3-5 个局部细节标注(各带简短中文解释)
- 4-6 个功能模块(各一行作用)
- 4-6 个工作原理流程节点(箭头连接)
- 材料/颜色渲染说明

标签保持短而清晰:`外壳`、`密封圈`、`控制芯片`、`传感器`、`电机`、`齿轮组`、`散热片`、`接口`、`电池`、`滤芯`、`支架`、`导流通道`。

**防幻觉**:不确定的内部结构标「⚠ 示意图」,绝不伪装精确。

**完成后,读取 [step-01-confirm.md](step-01-confirm.md) 继续。**
