# Step 0: 防幻觉分析

先建工作目录、锁定内容,再谈出图。这一步的产物是后续提示词的「事实底座」。

## 0.1 建工作目录

- 用源内容的标题或主题生成 slug(kebab-case,从标题提取关键词、去停用词、限 72 字符)。
- 在**用户当前工作目录**下建:
  ```
  infographic/{slug}/
  ├── source-{slug}.md      # 源内容存档
  ├── analysis.md           # 分析结论
  ├── structured-content.md # 防幻觉的结构化模块
  ├── refs/                 # 参考图(Step 2 用)
  └── prompts/              # 提示词(Step 2 用)
  ```

## 0.2 读取源内容

- **链接** → 用 webReader / fetch 抓取正文,存入 `source-{slug}.md`。
- **本地文档** → Read 后存入 `source-{slug}.md`。
- **对话/粘贴内容** → 直接整理存档。
- **只有主题、无源内容** → 先起草大纲并明确标注「⚠ AI 草拟,需用户确认」,不要伪装成既成事实。

## 0.3 写 analysis.md

至少包含:
- **主题**:一句话概括
- **受众**:技术 / 管理层 / 大众
- **内容类型**:技术架构 / 数据指标 / 时间流程 / 对比选型 / 概览汇报 / 其他(决定 Step 1 的参数推荐)
- **复杂度**:高 / 中 / 低
- **推荐参数**:初步建议的 layout / style / aspect(Step 1 再确认)

## 0.4 写 structured-content.md(防幻觉核心)

把内容拆成 **4-7 个视觉模块**,每个模块写清:

- **模块标题**
- **关键概念**:一句话
- **精确数据点**:数字 / 术语 / 引用,**必须来自源内容**,标注出处
- **视觉处理**:图标 / 图表 / 对比 / 强调方式
- **文本标签**:图片上要显示的中文文字(原样保留,Step 2 直接进提示词)

**防幻觉铁律**:任何源内容里没有的数据点 → 标注「⚠ 待确认」,绝不让它进入提示词。宁可少写一个模块,也不编一个数字。

**完成后,读取 [step-01-confirm.md](step-01-confirm.md) 继续。**
