# Step 0: 收集信息

## 目标

收集用户的问题描述和可用诊断资源，为后续分析做准备。

## 流程

### 0.1 询问用户信息（不可跳过）

用**单次** AskUserQuestion 同时收集两份信息：

**问题 1 — header: "问题描述"**
- 选项：`[输入问题描述]` — 传输方向、文件大小、速度差异等
- 选项：`[跳过，稍后补充]`

**问题 2 — header: "代码路径"**
- 选项：`[提供内核代码路径]` — Qualcomm 内核源码根目录路径
- 选项：`[跳过，不需要]` — 仅做日志分析，不检查代码

### 0.2 收集完成后

1. 记录用户提供的问题描述和代码路径
2. 自动扫描当前工作目录查找 `*.log` 文件（日志默认在当前目录）
3. 如果提供了代码路径 → 验证目录存在并记录
4. 列出找到的日志文件请用户确认

---

完成后，读取 `workflows/step-01-extract-timing.md` 继续。
