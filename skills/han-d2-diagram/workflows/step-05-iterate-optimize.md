# Step 5: 迭代优化

根据用户反馈和编译结果，迭代优化图表。

## 优化维度

### 布局优化

问题：节点重叠、连接交叉、布局不均衡

方法：
1. 更换布局引擎：复杂架构图 → `elk`，简单流程图 → `dagre`
2. 调整方向：层级结构 → `direction: down`，时序 → `direction: right`
3. 添加容器分组：
```d2
frontend: {
  label: 前端层
  web: Web应用
  mobile: 移动应用
}
```

### 样式优化

问题：颜色不协调、重点不突出、风格不统一

方法：
1. 语义化颜色：成功路径→绿，错误路径→红，决策节点→橙
2. 强调重点：
```d2
critical: 关键路径 {
  style.stroke-width: 3
  style.stroke: "#FF0000"
}
```
3. 用变量管理颜色：
```d2
colors: {
  primary: "#2196F3"
  danger: "#F44336"
}
node: {
  style.fill: colors.primary
}
```

### 可读性优化

问题：标签过长、文字过小、信息过载

方法：
1. 简化标签（4-6 字）
2. 调整字体大小：`style.font-size: 24`
3. 拆分复杂图表（单图节点 > 50 → 拆分为子图）

### 内容优化

问题：信息缺失、关系错误、冗余节点

方法：
1. 补充缺失连接和关键节点
2. 纠正连接方向（A→B vs B→A）
3. 删除不重要的中间节点

## 用户反馈信号识别

- "太挤了"、"重叠了" → 布局优化
- "颜色不好看"、"重点不突出" → 样式优化
- "字太小"、"太复杂" → 可读性优化
- "少了XX"、"关系错了" → 内容优化
- "好了"、"可以了"、"搞定" → 进入 Step 6

## 快速修复

| 问题 | 修复 |
|------|------|
| 节点重叠 | `--pad 50` + `-l elk` |
| 连接交叉 | 换 `direction: right` |
| 标签太长 | 缩短为 4-6 字 |
| 颜色单调 | 添加 2-3 种主色 |
