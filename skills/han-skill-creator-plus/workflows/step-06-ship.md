# Step 6: 最后一公里（安装与自测）

文件生成完不算交付——装上、跑通、确认能触发，才算闭环。本步骤补官方 skill-creator 不覆盖的"落地"环节。

## 6.1 安装到正确位置

用 AskUserQuestion 确认安装范围：
- **项目级**：`.claude/skills/<name>/`（仅当前项目可见）
- **用户级**：`~/.claude/skills/<name>/`（所有项目可见，推荐）

复制整个 skill 目录到目标位置。

## 6.2 校验名字一致

**目录名 == SKILL.md frontmatter 的 name**，不一致则触发不了。用 `grep '^name:' <path>/SKILL.md` 核对。

## 6.3 重启会话

Claude Code 在会话启动时加载 skill 元数据（name + description）。新装的 skill 必须**新开会话**才会被识别——当前会话看不到它属正常，别误判为安装失败。

## 6.4 自测触发

在新会话里，用**接近真实需求的具体语句**测试，别用太泛的话：
- ✅ 好："每天把客户报价邮件整理成 客户名|金额|日期 一行"（场景 + 输入 + 输出都具体）
- ❌ 差："帮我处理邮件"（太泛，本来就不会精准触发）

确认触发后跑一遍，看产出是否符合 Step 0 确认门里的"需求小结"。不符合就回 Step 3 迭代。
